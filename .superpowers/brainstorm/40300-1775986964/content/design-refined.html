Abadikan Landing — Revised Design (Post Deep Research)

Based on: linear.app (CSS sticky, no GSAP), stripe.com (vanilla JS), vercel.com (Motion), Codrops sticky grid

Key finding yang mengubah approach

Linear.app tidak pakai GSAP — mereka pakai CSS position: sticky + IntersectionObserver. Stripe juga pakai vanilla CSS transition. Ini berarti kita bisa dapat efek yang sama (bahkan lebih baik) dengan bundle ~3x lebih kecil. GSAP hanya untuk Gallery horizontal scroll di desktop. Semua section lain: pure CSS + IntersectionObserver yang dienhance.

LandingHowItWorks
CSS Sticky IO Enhanced
Layout baru: 2-column. Kiri: heading + overline + CTA button — sticky saat steps discroll. Kanan: 5 step block dengan connecting line vertikal SVG di antara step dots.

LandingGallery — Horizontal Pinned
GSAP Desktop CSS Mobile
Desktop (≥1024px): Section pin ke viewport. Scroll vertikal dikonversi ke horizontal movement. Tiap tema tampil full-width, judul + info di bawah tema. Progress dots di kanan bawah.
// GSAP ScrollTrigger — desktop only via matchMedia
gsap.to('.themes', { xPercent: -100 * (n-1), ease: 'none',
  scrollTrigger: { pin: true, scrub: 1, snap: 1/(n-1) } })
// Lazy loaded — hanya desktop, tidak diload di mobile
Mobile: CSS scroll-snap horizontal carousel native. Swipe biasa. Touch-friendly. Tidak ada GSAP.
